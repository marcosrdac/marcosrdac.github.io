Usando um teclado internacional

Posted on Tue 12 January 2021 in blog • 4 min read

Comecei a investir uma parte do que ganho no meu próprio ambiente de trabalho este ano. Passo a passo, comprei apoios de pulso para teclado e mouse, bem como apoios para os pés. Aumentei a altura do monitor com alguns livros e caixas, de maneira bem improvisada… E tudo isso melhorou bastante a saúde dos meus músculos dos braços, bem como da minha coluna. Desde então planejei a aquisição de um teclado mecânico. Finalmente, foi este o mês em que ele arrumou espaço na minha escrivaninha.

O meu teclado possui 82 teclas, incluindo as de função (F1, …, F12) e excluindo o teclado numérico, o que é bom para quem precisa usar o mouse com alguma frequência, pois este pode ficar mais próximo do centro do teclado, onde as mãos geralmente ficam. Recomendo essa configuração para programadores, mas não o faço caso você acredita sentir falta dos números em grade no seu teclado, porém.

Um detalhe é que este é o primeiro teclado que usei com layout internacional para a língua inglesa. Eu sabia disto quando o comprei, mas aceitei o desafio de lidar com esta característica, e este post é possui duas soluções que encontrei.

Solução para quem tem pressa

A solução mais direta para o layout diferente do que você deseja é a de fuçar as configurações do seu sistema operacional e mudar o layout de teclado usado mesmo para o da sua língua — o que significa escolher o modelo ABNT2 se você é brasileiro. Isto mudará a forma que o seu sistema interpreta os comandos do teclado, e tem como desvantagem que no geral a tinta nas teclas pressionadas não vai representar o está sendo digitado. De todo modo, seu teclado vai se comportar como um teclado brasileiro: c com cedilha do lado do L, acentos no lugar onde colchetes estão pintados, aspas simples e duplas logo abaixo do Esc.

Há muito tutorial de execução destas mudanças na internet. Num Linux, a solução em linha de comando é normalmente algo assim:

setxkbmap -model abnt2 -layout br

Mas se você também quiser trocar o caps-lock com o esc, basta adicionar esta opção:

setxkbmap -model abnt2 -layout br -option caps:swapescape

Adicionar um destes comandos no arquivo ~/.xprofile torna esta mudança permanente para as próximas sessões.

Solução para quem programa ou escreve em inglês

Passei um tempo na primeira solução, mas acabei me forçando um pouco a usar a versão internal do teclado, do jeito que ele foi feito. E aí descobri algo incrível: tanto programar quanto escrever textos em inglês é melhor no layout internacional, que é o estadunidense (us). O dedo mindinho da mão direita, que eu usava pouco no layout brasileiro, tem uma função geral no layout americano: usar acentuações. A única letra que você digita com ele é a letra P, pois do resto ele e só ele digita [], {}, /, ?¿, :;, '", etc. Como no teclado nacional a gente acaba usando ele primariamente para escrever P e Ç é mais difícil generalizar essa coisa dos símbolos especiais em um dedo em nossas mentes. Mais do que isto, a falta da tecla cedilha reorganiza essas teclas em posições muito mais interessantes. E este detalhe muda tudo quando estamos escrevendo.

Mas Marcos, não tem cedilha!

De fato, por padrão você não vai conseguir por cedilha ou acentuar as palavras. Daí vem a minha dica: use a versão do layout internacional com dead keys. Desta forma, as seguintes combinações geram símbolos comuns no português:

Pouco tempo de uso do layout o tornou padrão para mim, tanto para escrever em inglês e programar, quanto para escrever em português: o layout é muito mais fácil para os primeiros casos, enquanto passa a ser equivalente no segundo

Usando o método da linha de comando, este layout é escolhido no Linux de forma similar ao anterior, mas usando a variante intl, com dead keys, do layout estadunidense:

setxkbmap -layout us -variant intl

Novamente, para trocar o Caps Lock com o Esc se usa:

setxkbmap -layout us -variant intl -option caps:swapescape

Espero que este post tenha sido útil para você. Tenha bom proveito com seu novo teclado!
